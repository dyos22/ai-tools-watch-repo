# AI-verktygsspaning — 31 maj 2026

Veckans spaning är ovanligt konkret: de stora modellerna tar paus i rampljuset medan verktygen *runt* dem exploderar i antal. Tre spår dominerar — videogenerering som börjar likna riktig filmproduktion, röst-AI som nu kan köras utan att data lämnar ditt kontor, och en hel flora av verktyg för att göra AI-assistenter som Claude Code mer pålitliga och smarta.

---

## Topp 3 att testa direkt

- **ViMax** — Skriv en idé eller ett manus och få tillbaka en hel video med konsekventa karaktärer och dramatisk uppbyggnad — inte ett kortklipp utan en berättelse.
- **Dograh** — En gratis, öppen röst-AI-plattform du kör på egna servrar: inga månadsavgifter, datan stannar hos dig.
- **OpenMemory** — Installera ett plugin som ger Claude Code (och Cursor, Copilot m.fl.) ett riktigt minne — ett som håller koll på *vad du beslutade* förra månaden, inte bara vad du frågade om igår.

## Skapa saker med AI

### ViMax
**Vad det är:** Tolv specialiserade AI-agenter samarbetar som ett virtuellt filmteam — regissör, manusförfattare, storyboardkonstnär, karaktärskonsult — för att förvandla en idé, roman eller ett manus till en sammanhängande video.

**Varför det är intressant:** Sora, Kling och Runway fungerar som en kamera du pekar och klickar på: du skriver en prompt, du får ett klipp. De håller inte ihop längre berättelser — karaktärerna ser annorlunda ut i nästa scen, handlingen glöms bort. ViMax löser detta genom att dela upp jobbet som ett riktigt filmteam: en agent skriver manuset, en annan planerar klippningen, en tredje ser till att huvudpersonen ser ut på samma sätt i varje scen. Stöder fyra lägen: idé-till-video, roman-till-video, manus-till-video, och "AutoCameo" där du lägger in ditt eget ansikte via ett foto.

**Buzz-nivå:** Håller på att bli en grej — ~8 400 GitHub-stjärnor, +1 500 nya bara den senaste veckan. Ingen officiell lansering ännu, aktiv utveckling.

**Testa:** [github.com/HKUDS/ViMax](https://github.com/HKUDS/ViMax) — kräver teknisk setup, men demovideorna på GitHub visar kapaciteten tydligt.

### Dograh
**Vad det är:** En öppen plattform för att bygga röst-AI som du kör på dina egna servrar — ett gratis alternativ till Vapi och Retell, de dominerande betaltjänsterna för röst-AI.

**Varför det är intressant:** ElevenLabs är fantastiskt för röster, men allting du säger och hör går via deras servrar och faktureras per minut. Dograh vänder på det: du väljer din egen röstsyntes, din egen AI-modell och kör alltihop lokalt. En visuell byggare låter dig sätta ihop röstflöden utan att skriva kod — som ett pratande Lego. Perfekt för företag med sekretessregler, eller för den som vill ha full kontroll utan löpande kostnader.

**Buzz-nivå:** Håller på att bli en grej — ~3 900 GitHub-stjärnor, +1 200 nya på en vecka. Lanserat 17 maj 2026.

**Testa:** [github.com/dograh-hq/dograh](https://github.com/dograh-hq/dograh) — startar med ett kommando, dokumentation på dograh.com.

## Gör Claude Code och andra AI-assistenter smartare

### OpenMemory
**Vad det är:** Ett plugin som ger Claude Code, Cursor, Windsurf och GitHub Copilot ett strukturerat, tidsstämplat minne — och som skiljer på vad du *beslutade* för tre veckor sedan mot vad du *ändrade dig om* förra månaden.

**Varför det är intressant:** Idag glömmer alla AI-assistenter allt mellan sessioner. Det "minne" som finns i t.ex. ChatGPT fungerar som en enkel texthög — det hittar det som *liknar* din nuvarande fråga men förstår ingen tidslinje. OpenMemory bygger istället ett kunskapsnätverk med tidsstämplar, så assistenten kan svara på frågor som "vilken databasstruktur valde vi i april, och har vi ändrat oss sedan dess?" Körs lokalt på din dator — inget lämnar dina servrar. Installeras direkt i Claude Code som ett MCP-plugin (ett standardiserat sätt att utöka Claude Code med nya förmågor).

**Buzz-nivå:** Håller på att bli en grej — 4 200 GitHub-stjärnor, uppmärksammad av välkända namn i ML-världen.

**Testa:** [github.com/CaviraOSS/OpenMemory](https://github.com/CaviraOSS/OpenMemory) — installeras med ett kommando direkt i Claude Code.

### oh-my-pi (omp)
**Vad det är:** En terminalbaserad AI-kodningsagent med inbyggd debugger, webbsökning, filhantering och stöd för 40+ AI-modeller — ett kraftfullt, öppet alternativ till Claude Code och Cursor.

**Varför det är intressant:** Cursor och Claude Code redigerar filer med ett textbaserat "diff"-format som ibland missar rätt ställe i koden, framför allt i stora filer. oh-my-pi använder ett annat system som eliminerar sådana fel och gör varje redigering snabbare och billigare. Inbyggd debugger (kan stega igenom kod rad för rad, som en riktig IDE), stöd för 40+ AI-modeller, och möjlighet att dela upp uppgifter i parallella deluppgifter. +2 000 nya stjärnor på en vecka är ett starkt signal om att det håller på att hittas av ett stort antal just nu.

**Buzz-nivå:** Håller på att bli en grej — ~8 800 GitHub-stjärnor, +2 000 den senaste veckan. Version 15.x med 400+ releaser sedan start, vilket tyder på extremt hög iterationshastighet.

**Testa:** [github.com/can1357/oh-my-pi](https://github.com/can1357/oh-my-pi) — installeras med npm, används med din befintliga API-nyckel.

### herdr
**Vad det är:** Ett terminalverktyg som fungerar som ett kontrollrum för att hantera och övervaka flera AI-kodningsagenter (Claude Code, Codex, m.fl.) parallellt — med färgkodad status i realtid.

**Varför det är intressant:** Allt fler kör numera *flera* AI-assistenter på gång samtidigt för att snabba upp jobbet. Men att hålla koll på dem liknar att stirra på fem terminalfönster utan att veta vilka som jobbar, väntar eller fastnat. Herdr ger dig en samlad vy: varje agent visas som en ruta med en färgpunkt — grön = jobbar, gul = väntar, röd = fastnat. Byggt i Rust, inga externa beroenden, funkar inuti tmux. Uppbyggt till stor del av de agenter det nu hanterar — ett litet bevis på konceptet.

**Buzz-nivå:** Börjar fångas upp — ~3 200 GitHub-stjärnor, +900 den senaste veckan.

**Testa:** [github.com/ogulcancelik/herdr](https://github.com/ogulcancelik/herdr) — laddas ned som en enda körbar fil, inga beroenden att installera.

### Harbor
**Vad det är:** Ett kommandoradsverktyg som med ett enda kommando (`harbor up`) sätter upp en komplett, färdigkonfigurerad lokal AI-stack — modellkörare, chattgränssnitt, bildskapare, röstchatt, vektordatabas och mer.

**Varför det är intressant:** Att sätta upp en lokal AI-miljö som ett alternativ till ChatGPT-/Claude-prenumerationen har historiskt krävt timmar av pyssling med konfigurationsfiler. Harbor kollapsar det till ett kommando. Tänk det som Spotify-appen för lokalt AI: allt är förkopplat och redo med 90+ tjänster (Ollama för modeller, ComfyUI för bilder, Open WebUI som chattgränssnitt m.m.). En "Harbor Boost"-funktion byter automatiskt till en reservmodell om din primärmodell är nere eller full.

**Buzz-nivå:** Börjar fångas upp — ~3 000 GitHub-stjärnor, aktiv v0.4.x-utveckling.

**Testa:** [github.com/av/harbor](https://github.com/av/harbor) — installeras med pip, startas med `harbor up`.

## Specialgrejer & experiment

### AgentQL
**Vad det är:** Ett frågespråk som låter AI-agenter interagera med webbsidor genom att beskriva *vad* de letar efter ("inloggningsknapp", "sökfält", "pris") istället för tekniska HTML-koder som slutar fungera varje gång en sajt uppdateras.

**Varför det är intressant:** Klassisk webbautomation slutar fungera ungefär var tredje månad när sajter gör om sin design — och kräver manuell fix varje gång. AgentQL löser problemet strukturellt: om knappen fortfarande heter "Logga in" fungerar frågan oavsett hur koden bakom ser ut. Jämfört med att låta AI titta på en skärmbild och gissa var man ska klicka är det snabbare och billigare. Tänk det som ett stabilt lim som håller ihop AI-agenter och valfri hemsida — utan att gå sönder vid varje redesign.

**Buzz-nivå:** Extremt tidigt — ~1 100 GitHub-stjärnor. Diskuteras alltmer som en ny infrastrukturkomponent för produktionsagenter.

**Testa:** [github.com/tinyfish-io/agentql](https://github.com/tinyfish-io/agentql) — Python-paket med dokumentation på [docs.agentql.com](https://docs.agentql.com).

### turbovec
**Vad det är:** En extremt snabb textsökmotor i Rust, baserad på en Google Research-algoritm accepterad vid ICLR 2026, som kan söka i stora textsamlingar utan en tidskrävande förberedelsefas.

**Varför det är intressant:** Det vanligaste verktyget för att söka i stora textsamlingar (FAISS) kräver att du "tränar" ett specialindex på *hela* datauppsättningen innan du kan börja söka — opraktiskt om man vill lägga till dokument löpande. turbovec behöver ingen träningsfas alls: lägg till enskilda dokument och sök direkt. 12–20% snabbare än FAISS på moderna chips. Möjliggör AI-söklösningar som kan uppdateras i realtid snarare än i schemalagda batcher. MIT-licens.

**Buzz-nivå:** Börjar fångas upp — ~950 GitHub-stjärnor, men +1 650 nya på en enda vecka, ett tecken på att det just nu hittas av många.

**Testa:** [github.com/RyanCodrai/turbovec](https://github.com/RyanCodrai/turbovec) — Python-paket med enkel API.

### Statewright
**Vad det är:** Ett verktyg som tvingar AI-agenter att följa ett definierat arbetsflöde — t.ex. "planera först, redigera sedan, testa sist" — och blockerar dem från att använda fel verktyg i fel fas.

**Varför det är intressant:** AI-agenter hoppar gärna direkt till att redigera kod utan att ha tänkt igenom problemet, eller kör tester när de borde läsa koden. Statewright låser varje fas: under planeringsfasen kan agenten bara läsa filer, inte ändra dem; under implementationsfasen kan den redigera men inte köra tester. I tester förbättrades slutresultaten från 2/10 till 10/10 lösta uppgifter med lokala AI-modeller när begränsningarna sattes på. Fungerar med Claude Code, Cursor och andra via det standardiserade MCP-protokollet.

**Buzz-nivå:** Extremt tidigt — 372 GitHub-stjärnor. Show HN publicerat 20 maj 2026.

**Testa:** [github.com/statewright/statewright](https://github.com/statewright/statewright) — plugin för Claude Code och Cursor.
