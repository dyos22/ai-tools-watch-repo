# AI-verktyg att hålla koll på — 7 juni 2026

## Inledning

Den här veckan domineras spaningen av ett tydligt mönster: AI-agenter är inte längre ett experiment — det handlar nu om *infrastrukturen runt dem*. Vad kostar det att köra dem? Hur håller man dem säkra? Hur ger man dem ett minne? Hur stoppar man dem från att göra fel? Tolv fynd den här veckan, varav flera direkt användbara för den som redan jobbar med Claude Code.

## Gör Claude Code och andra AI-assistenter smartare

### Compound Engineering Plugin (EveryInc)

**Vad det är:** Ett officiellt tillägg till Claude Code, Cursor och Codex med 40+ specialiserade mini-agenter, 30+ kommandon och 35+ färdiga arbetsflöden som täcker hela mjukvaruutvecklingsprocessen — från arkitekturreview till release notes.

**Varför det är intressant:** Claude Code är bra på att koda, men saknar djupet för domänspecifika arbetsflöden. Compound Engineering Plugin ger varje del av processen sin egen specialiserade agent som i sin tur gör nästa del enklare. Det är skillnaden mellan en generalist-assistent och ett helt team. Varje agent är tränad på ett specifikt jobb: PR-feedback, testning, arkitekturanalys. Jämfört med GitHub Copilots tillägg är orkestreringen på en helt annan nivå.

**Buzz-nivå:** Håller på att bli en grej — lanserades 30 maj 2026 med 18–20 000 GitHub-stjärnor.

**Testa:** [github.com/EveryInc/compound-engineering-plugin](https://github.com/EveryInc/compound-engineering-plugin) — installera som plugin i Claude Code och kör `/help` för att se hela kommandobibliotekat.

### Salt Code (Salt Security)

**Vad det är:** Ett säkerhetslager som sitter *inuti* AI-kodningsassistenter och blockerar säkerhetshål i realtid — innan koden ens genereras.

**Varför det är intressant:** Antalet säkerhetsproblem i AI-genererad kod sexdubblades under 2025–2026. Traditionella säkerhetsscanners tittar på koden *efteråt*; Salt Code pluggar in som ett MCP-konfigurationslager i Claude Code, Cursor, Copilot, Windsurf med flera och definierar säkerhetspolicyer som gäller direkt när koden genereras. Ingen annan lösning sitter på exakt den platsen i flödet.

**Buzz-nivå:** Börjar fångas upp — lanserades 1 juni 2026, täckt av IT Security Guru och SiliconANGLE.

**Testa:** [prnewswire.com/salt-code](https://www.prnewswire.com/news-releases/salt-security-launches-salt-code-302786506.html) — konfigureras som MCP-server i Claude Code-installationen.

### Composio Agent Orchestrator

**Vad det är:** En AI-agent som hanterar *andra* AI-agenter — startar parallella kodningsagenter i separata arbetsytor och sköter CI-fel, sammanslagningar och code reviews automatiskt.

**Varför det är intressant:** Claude Code löser en uppgift i taget. Composio vänder på det: en orkestreringsagent läser hela backlogen, bryter ner uppgifter, tilldelar dem och fattar beslut om nästa steg. Du reviewar färdiga pull requests — allt däremellan sköts automatiskt. Stöder Claude Code, Codex, Aider och OpenCode. Skiljer sig från liknande verktyg (som DeepSWE) genom att fokusera på hela flödet runt CI och pull requests, inte bara på att generera patchar.

**Buzz-nivå:** Börjar fångas upp — refereras i Augment Codes guide "9 öppna agentorkestrerare 2026".

**Testa:** [github.com/ComposioHQ/agent-orchestrator](https://github.com/ComposioHQ/agent-orchestrator)

## Verktyg, automation & infrastruktur

### Headroom

**Vad det är:** Ett öppet bibliotek som komprimerar allt en AI-agent läser — kodfiler, loggar, sökresultat, dokumentation — med 60–95 % innan det skickas till AI-modellen, och hanterar detta transparent utan att svaren försämras.

**Varför det är intressant:** Att köra AI-agenter på stora kodprojekt eller dokument kan kosta tiotals kronor per körning, eftersom priset betalas per ord agenten processerar. Headroom löser detta med tre specialiserade algoritmer: en för strukturerad data som JSON och tabeller, en för källkod som förstår kodens uppbyggnad och tar bort det som verkligen är redundant, och en som lagrar originalinnehållet lokalt och skickar en sammanfattning till AI:n. Fungerar som bibliotek, som proxy eller som en MCP-server — passar in utan att ändra befintlig setup.

**Buzz-nivå:** Extremt tidigt men exploderade precis — 13 300 av 16 000 totala GitHub-stjärnor tillkom under en enda vecka (5–7 juni 2026).

**Testa:** [github.com/chopratejas/headroom](https://github.com/chopratejas/headroom) — `pip install headroom` och koppla till befintlig agentsetup.

### SMFS — Supermemory Filesystem

**Vad det är:** Ett filsystem byggt speciellt för AI-agenter, där vanliga sökkommandon ger svar baserade på *mening* snarare än exakta textmatchningar.

**Varför det är intressant:** Traditionella filsystem förstår inte innebörden i ett dokument — de matchar exakt text. SMFS låter agenten söka och få tillbaka rätt dokument baserat på vad de *handlar om*, inte exakt hur de är skrivna. Hanterar automatiskt PDF:er, videofiler, ljud och bilder. Tillverkaren hävdar 55 % lägre kostnad och högre träffsäkerhet jämfört med klassisk vektorbaserad dokumenthämtning (RAG). Kan köras var som helst, inklusive serverlösa miljöer.

**Buzz-nivå:** Börjar fångas upp — lanserades sent maj 2026, moderbolaget Supermemory har 25 000+ stjärnor.

**Testa:** [github.com/supermemoryai/smfs](https://github.com/supermemoryai/smfs)

### Sim (Sim Studio)

**Vad det är:** En visuell drag-and-drop-plattform för att bygga och koppla ihop AI-agenter med 1 000+ integrationer — utan att skriva kod.

**Varför det är intressant:** Langflow och Dify finns sedan tidigare, men Sim positioneras som modernt och "2026-native": realtidssamarbete med live-cursor (flera personer bygger samma agentflöde), rollbaserad åtkomst och en inbyggd Copilot som bygger och kopplar ihop block från naturligt språk — du beskriver vad du vill och den ritar upp flödet. YC-backat startup. Stöder alla stora AI-modeller.

**Buzz-nivå:** Börjar fångas upp — YC-backed, listat som ett av de starkast rekommenderade öppna agentverktygen i maj–juni 2026.

### QwenPaw

**Vad det är:** En personlig AI-assistent med öppen källkod som lever på din egna server och nås via Telegram, Discord, WeChat, Slack, iMessage och 15 andra plattformar.

**Varför det är intressant:** Skillnaden mot att använda ChatGPT eller Claude direkt: QwenPaw minns allt mellan sessioner, kör på din egna hårdvara (inga data till tredjepartsmoln om du inte vill), och med version 1.1.10 (1 juni 2026) kan den nu spawna egna sub-agenter — delegera delar av en uppgift till specialiserade mini-agenter och sammanfoga resultaten. Beskriv ett mål, gå och sov, vakna till en prototyp.

### Brief

**Vad det är:** Ett verktyg för team som samlar alla produktbeslut och intentioner på ett enda ställe, tillgängligt för både människor och AI-agenter via chat, Slack, CLI och MCP.

**Varför det är intressant:** AI-agenter är duktiga på att koda men vet inte *varför* något ska byggas på ett visst sätt — de saknar tillgång till besluten som togs för sex månader sedan. Brief skapar en "levande kunskapsbas" av produktintentioner och kopplar den direkt till agenternas kontext via MCP-protokollet. Det är en ny kategori: inte ett doc-verktyg, inte en wiki, utan ett "kontextlager" som agenter kan slå upp i realtid.

**Buzz-nivå:** Börjar fångas upp — fick Product of the Day på Product Hunt 2 juni 2026.

**Testa:** [producthunt.com/products/brief-10](https://www.producthunt.com/products/brief-10/awards) — gratis att börja, MCP-integration ingår.

### Bloom

**Vad det är:** En tjänst som förvandlar ett varumärkes grafiska identitet — sajt, Figma-filer, presentationer, sociala medier — till ett anropbart API som AI-agenter kan använda direkt.

**Varför det är intressant:** AI-agenter kan generera text men inte varumärkeskonsekvent grafik — de vet inte vilka exakta färger, typsnitt och bildspråk ett varumärke använder. Bloom löser detta genom att exponera brand-generering som ett verktyg: Claude Code, Cursor eller ChatGPT kan anropa Bloom via MCP och få tillbaka grafik som faktiskt håller stil. YC-backat, prissatt från $20/mån med API och MCP inkluderat.

**Buzz-nivå:** Extremt tidigt — YC Launch ~2 juni 2026.

**Testa:** [trybloom.ai/mcp](https://www.trybloom.ai/mcp/) — MCP-server finns, kräver konto.

### Fingerprint Automation Intelligence API

**Vad det är:** Ett API som identifierar AI-assistenter och automatiserade agenter bland besökarna på en webbplats — utan att behöva köra JavaScript i en webbläsare.

**Varför det är intressant:** Traditionell bot-detektering kräver att ett skript körs i besökarens webbläsare. AI-agenter använder ingen webbläsare — de hämtar sidor direkt som program. Fingerprint täpper blind spot:en genom att analysera trafiken på HTTP-nivå: IP-intervall, DNS-verifiering och mönsterigenkänning. I takt med att AI-agenter utgör en allt större andel av webbtrafiken blir detta infrastruktur, inte nisch.

**Buzz-nivå:** Börjar fångas upp — täckt av SC Media och SiliconANGLE vid Preview GA-lansering 1 juni 2026.

**Testa:** [businesswire.com/fingerprint-automation](https://www.businesswire.com/news/home/20260601158287/en/Fingerprint-Launches-Automation-Intelligence-API-and-AI-Assistant-Detection) — Preview API, kontakta för access.

### FlowAI (Itential)

**Vad det är:** En plattform för att bygga och köra AI-agenter mot nätverks- och IT-infrastruktur, med inbyggd styrning, rollbegränsningar och fullständig granskningslogg vid varje åtgärd.

**Varför det är intressant:** Generella agentverktyg (LangChain, Dify, Sim) saknar mekanismer för att stoppa en agent från att göra irreversibla saker i produktionssystem. FlowAI lägger ett styrningslager ovanpå: mänskligt godkännande krävs vid kritiska steg, varje åtgärd loggas med vem som godkände den, och rollbegränsningar definierar exakt vad en agent får och inte får göra. Inkluderar FlowMCP Gateway för externa MCP-verktyg. Validerat i sex månader i telekom, finans och utilities.

**Buzz-nivå:** Börjar fångas upp — lanserades på Cisco Live US 1 juni 2026, GA startar 1 juli 2026.

**Testa:** [itential.com/flowai](https://www.itential.com/flowai/) — kontakta för early access.

## Specialgrejer & experiment

### Mellum2 (JetBrains)

**Vad det är:** En liten, snabb och öppen AI-modell specialiserad på källkod — designad för att vara "routing-hjärnan" i system med flera AI-modeller snarare än att själv lösa allt.

**Varför det är intressant:** Det finns ett växande problem med AI-agenter som alltid skickar varje fråga till den dyraste modellen (Claude Opus, GPT-4o) — även när det räcker med något enklare. Mellum2 är tränad på 10,6 biljoner tokens med fokus på kod och är specialiserad på att avgöra *vilken* modell som ska lösa en uppgift, hämta rätt dokumentation, och köra enkla kodkompletteringar till en bråkdel av kostnaden. Mer än 2x snabbare mot jämförbara modeller. Öppen källkod (Apache 2.0), från JetBrains — skaparna av IntelliJ och PyCharm.

**Buzz-nivå:** Börjar fångas upp — lanserades 1 juni 2026, täckt av The New Stack och AIWeekly inom 48 timmar.

**Testa:** [huggingface.co/JetBrains/mellum2](https://huggingface.co/blog/JetBrains/mellum2-launch) — ladda ner och kör lokalt via Ollama eller direkt på HuggingFace.
