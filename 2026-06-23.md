# AI-verktyg att hålla koll på — 23 juni 2026

## Inledning

Veckans spaning bekräftar ett skifte som pågått i ett par veckor: fokus är inte längre att *prova* AI-agenter utan att *driftsätta* dem ordentligt. Tre teman dominerar — agenter i molnet som triggas av Slack och GitHub och levererar tillbaka en pull request, verktyg för att orkestrera och övervaka flera parallella agentsessioner, och ny infrastruktur för att mäta hur bra agenter egentligen presterar och skydda känslig data längs vägen.

---

## Topp 3 att testa direkt

- **Replicas** — Kör Claude Code eller valfri kodningsagent i en isolerad moln-VM, trigga den från Slack eller GitHub, och få tillbaka en färdig pull request.

- **Agent Deck** — En terminal-kontrollpanel som visar tio parallella Claude Code-sessioner i ett enda fönster — med kostnadsöversikt och Telegram-notiser när en agent fastnar.

## Gör Claude Code och andra AI-assistenter smartare

### Agent Deck

**Vad det är:** En terminal-app (ungefär som ett kontrollcenter) som visar alla dina parallella Claude Code-, Gemini CLI- eller Codex-sessioner i ett enda fönster med realtidsöverblick över vad de gör och vad de kostar.

**Varför det är intressant:** I dag jonglerar tunga Claude Code-användare fem–tio terminalflikar manuellt och tappar lätt koll. Agent Deck löser det — plus en bit till: det delar en gemensam "serveranslutning" (MCP socket pool) mellan alla sessioner, vilket sänker minnesanvändningen med 85–90 %. En inbyggd "Conductor" kan ta emot instruktioner via Telegram eller Slack och skicka dem vidare till rätt agent. Ingen konkurrent hanterar multi-session MCP-pooling på det här sättet.

**Buzz-nivå:** Börjar fångas upp — 345 GitHub-stjärnor, senaste release v1.9.73 den 21 juni 2026.

**Testa:** https://github.com/asheshgoplani/agent-deck — Ladda ner Go-binären och peka den mot dina befintliga Claude Code-projektmappar.

### Zot

**Vad det är:** En ultralätt kodningsagent (en enda nedladdningsbar fil, inget installationssteg) som låter dig grena ut konversationen — ungefär som att skapa en git-branch men för agentkörningar.

**Varför det är intressant:** Claude Code och liknande agenter går alltid rakt fram: ångrar du en strategi måste du börja om från noll. Zots `/tree`-kommando låter dig backa till valfri tidigare punkt och pröva ett annat spår utan att förlora det du redan gjort. Stöder dessutom 10+ LLM-leverantörer inklusive lokala modeller via Ollama — inget prenumerationskrav.

**Buzz-nivå:** Extremt tidigt — 272 GitHub-stjärnor, Show HN-diskussion juni 2026, 245 releaser på kort tid.

**Testa:** https://github.com/patriceckhart/zot — Ladda ner binären och kör `zot` i din projektmapp.

### agent-eval (Hugging Face)

**Vad det är:** Ett CLI-verktyg från Hugging Face som mäter *hur väl* en kodningsagent arbetar — inte bara om slutresultatet blev rätt, utan också tokenkostnad, tidsåtgång och felfrekvens längs hela vägen.

**Varför det är intressant:** Det saknas i dag ett neutralt sätt att jämföra "är Claude Code bättre än Gemini CLI för den här typen av uppgifter?" agent-eval ger ett kvantitativt svar som tar hänsyn till *processen*, inte bara slutnoten. Du ser exakt var en agent slösar pengar eller fattar onödiga beslut — och kan byta ut en dyr modell mot en billigare med ögonen öppna.

**Buzz-nivå:** Börjar fångas upp — lanserat 18 juni 2026, rapporterat av InfoQ och TechJack Solutions.

**Testa:** https://huggingface.co/blog/is-it-agentic-enough — Bloggpost med länk till verktyget och exempelnotebooks.

## Verktyg, automation & infrastruktur

### Replicas

**Vad det är:** En molntjänst (Y Combinator-backat startup) som kör Claude Code, Codex eller valfri kodningsagent i en isolerad virtuell dator i molnet — du triggar den via Slack, Linear, GitHub Issues eller ett API, och den levererar en pull request klar för granskning.

**Varför det är intressant:** Lokala agenter låser upp din dator och missar din faktiska CI-miljö. Replicas ger agenten en riktig dev-sandbox med dina egna API-nycklar, utan att du behöver lämna Slack. Inga installationssteg — fungerar med vilken kodningsagent som helst. Kunder rapporterar att 30 % av deras pull requests nu levereras automatiskt via tjänsten.

**Buzz-nivå:** Håller på att bli en grej — 221 upvotes på Product Hunt, 20+ YC-startups som kunder (bl.a. Mintlify och Composio).

**Testa:** https://www.replicas.dev/ — Kostnadsfri testperiod finns; demo-registrering på hemsidan.

### Cloudflare Temporary Accounts

**Vad det är:** En ny funktion i Cloudflares byggverktyg (Wrangler v4.102.0) som låter en kodningsagent deploya en webbapplikation helt utan mänsklig inloggning — deploymentet lever i 60 minuter och kan sedan "claimas" permanent av ägaren.

**Varför det är intressant:** AI-agenter som ska bygga och testa en webbsida kraschar i dag på inloggningsväggar. Den nya `--temporary`-flaggan bryter mönstret: agenten bygger, deployer och verifierar utan avbrott och överlämnar sedan en URL att godkänna. Det löser det konkreta hindret för en sann "kod → live-sida → test"-loop utan mänsklig inblandning i varje steg.

**Buzz-nivå:** Börjar fångas upp — annonserat 19 juni 2026 på Cloudflare Blog, uppmärksammat av Simon Willison (inflytelserik AI- och webbkommentator) den 21 juni.

**Testa:** https://blog.cloudflare.com/temporary-accounts/ — Uppdatera Wrangler och kör `wrangler deploy --temporary`.

### Xata Agent

**Vad det är:** En AI-agent som kopplas till din PostgreSQL-databas och självständigt övervakar, diagnostiserar och föreslår konkreta SQL-åtgärder 24/7 — stöder Amazon RDS, Aurora och Google Cloud SQL.

**Varför det är intressant:** Befintliga övervakningsverktyg som Datadog eller pganalyze visar vad som är fel men kräver att en expert tolkar och agerar. Xata Agent stänger loopen: den ser ett problem, förklarar det i klartext och föreslår ett SQL-kommando att köra — utan att en databasexpert behöver vakta dashboards. Öppen källkod och kan köras helt lokalt.

**Buzz-nivå:** Börjar fångas upp — ~785 GitHub-stjärnor, diskuterat på Hacker News (juni 2026).

**Testa:** https://github.com/xataio/agent — Kör `npx xata-agent` och ange din Postgres-anslutningssträng.

### Flue (Astro-teamet)

**Vad det är:** Ett ramverk för att bygga och deploya egna AI-agenter — från Astro-teamet (kända för en populär webbplatsbyggare) — som fungerar på Node.js, Cloudflare Workers och GitHub Actions utan att du ändrar ett enda kodrad.

**Varför det är intressant:** De flesta agentramverk (LangGraph, Mastra) kräver i praktiken att du väljer en molnleverantör och stannar där. Flue är "runtime-agnostiskt": du skriver agentlogiken en gång och väljer körplattform sedan. Astro-teamet är känt för att producera väldesignade, lättanvända verktyg — det ger projektet mer trovärdighet än ett genomsnittligt nylanserat GitHub-repo.

**Buzz-nivå:** Håller på att bli en grej — 6 447 GitHub-stjärnor, +1 413 senaste veckan, juni 2026.

**Testa:** https://github.com/withastro/flue — `npm create flue@latest` sätter upp ett startprojekt på tre minuter.

### microsoft/presidio

**Vad det är:** Ett öppen källkods-ramverk som automatiskt hittar och maskerar känslig personinformation (personnummer, namn, kreditkortsnummer, e-postadresser) i text och bilder *innan* de skickas till en AI-modell — 2026 tillkom en dedikerad pipeline för att sanitera AI-promptar.

**Varför det är intressant:** Med agenter som nu bearbetar verklig användardata är dataskydd ett infrastrukturproblem, inte ett tillval. Presidio körs helt lokalt (inget lämnar nätverket), stöder reversibel anonymisering (agentens svar kan "avmaskeras" i efterhand), och är det enda mogna open source-alternativet i sin klass. Avgörande för alla som hanterar patientjournaler, kontrakt eller HR-data via AI.

**Buzz-nivå:** Börjar fångas upp specifikt för agentanvändning — 9 473 GitHub-stjärnor, +827 senaste veckan.

**Testa:** https://github.com/microsoft/presidio — `pip install presidio-analyzer presidio-anonymizer`, sedan tre raders Python för att sanitera din första text.

### Kanban-Code

**Vad det är:** En Kanban-tavla (som Trello, men för kodningsagenter) där varje kort är en Claude Code-session kopplad till en egen git-gren och terminal — korten rör sig automatiskt från "att göra" till "klart" medan agenten arbetar.

**Varför det är intressant:** Att driva parallella Claude Code-sessioner i dag är ostrukturerat — det finns ingen naturlig vy för att se "vad pågår, vad är klart, vad kräver min uppmärksamhet?". Kanban-Code ger just den vyn och skickar push-notiser till telefonen när agenten behöver en människa i loopen. Varje session isoleras i sin egna git-branch (ingen risk för att agenter skriver över varandra). AGPL-3.0 och gratis.

**Buzz-nivå:** Extremt tidigt — 28 GitHub-stjärnor, Hacker News-diskussion (item #47573483) juni 2026.

**Testa:** https://github.com/langwatch/kanban-code — `npm install`, en konfigurationsfil, och du är igång.

## Specialgrejer & experiment

### VT Code

**Vad det är:** En kodningsagent byggd i programspråket Rust (känt för prestanda och säkerhet) med inbyggd *semantisk* kodförståelse — agenten förstår kodens struktur (klasser, funktioner, beroenden) snarare än att bara söka i text, plus OS-nativ sandboxning och stöd för 21 LLM-leverantörer.

**Varför det är intressant:** Claude Code och Aider söker i kod ungefär som avancerad textsökning. VT Code analyserar istället kodens faktiska struktur — som ett familjeträd av komponenter — vilket ger mer precisa svar på frågor som "vad beror X på?" och "var används den här funktionen?". OS-nativ sandboxning ger dessutom starkare säkerhetsgränser än processbaserade alternativ.

**Buzz-nivå:** Extremt tidigt — tre separata Show HN-inlägg på Hacker News (senast april 2026), aktivt underhållet.

**Testa:** https://github.com/vinhnx/VTCode — Kräver Rust-toolchain: `cargo install vtcode`.

---

### Skribe

**Vad det är:** En lokal Markdown-texteditor (kör på din dator, inget moln) med en inbyggd AI-granskningspartner — AI:n kan kopplas till Claude, Codex eller valfri kompatibel modell och lämnar kommentarer och förslag *utan att din text lämnar datorn*.

**Varför det är intressant:** Notion AI, Google Docs AI och liknande tjänster lagrar din text i deras moln. Skribe separerar dokumentet (lokalt, alltid ditt) från granskningsflödet (temporärt, ingen lagring) och stöder vilken modell som helst via ett standardiserat API. Relevant för känsliga texter som kontrakt, personliga anteckningar eller affärsdokument där du inte vill att innehållet ska träna andras modeller.

**Buzz-nivå:** Extremt tidigt — nytt verktyg, lyft fram i Ben's Bites juni 2026.

**Testa:** https://www.skribe-editor.com/ — Nedladdning direkt från webbplatsen, ingen registrering krävs.
