# AI-verktygsspaning — 2026-07-05

## Inledning

Veckans tema är tydligt: agenter som tar hand om sig själva. Tre mönster rör sig snabbt just nu — (1) verktygen för att köra *fler* agenter parallellt mognar snabbt, (2) infrastrukturen för att agenter ska kunna *prata med varandra* börjar ta form, och (3) säkerhetssidan av agentarbete — sandboxar, scanning, design-by-security — tar fart. Spaningen samlade in 21 fynd totalt, varav 13 klarade uteslutningslistan (~190 tidigare täckta verktyg) och kvalitetsfiltrering.

## Topp 3 att testa direkt

1. **Craft Agents OSS** — Polerad skrivbordsapp för att hantera flera AI-kodningsagenter parallellt med realtids-streaming och GitHub-integration — ingen terminal krävs.
2. **GNHF** — Sätt dina kodagenter att arbeta autonomt natten igenom; varje lyckad iteration committeras automatiskt och agenten återupptar där den slutade.

## Gör Claude Code och andra AI-assistenter smartare

### AMA2
**Vad det är:** En meddelandetjänst byggd för AI-agenter — tänk Slack, men där agenterna är fullvärdiga deltagare som minns vad som sades i förra sessionen.

**Varför det är intressant:** Problemet med att använda Claude Code eller Cursor med Slack och Discord är att agenten måste läsa hela konversationshistoriken från noll varje gång. AMA2 löser det med persistent kontext: agenten har sin egna identitet, kommer ihåg vad den gjort och kan till och med ta initiativ och kontakta dig. Det stöder dessutom de nya standardprotokollen för hur agenter kommunicerar med varandra (MCP och A2A), vilket gör det till en naturlig nav för multi-agent-arbete.

**Buzz-nivå:** Börjar fångas upp — Show HN-post på Hacker News (1 juli 2026) med aktiv diskussion; hittades oberoende av alla tre spaningens subagenter.

**Testa:** https://ama2.me/ — Skapa konto, koppla Claude Code via den inbyggda MCP-servern.

### GNHF (Good Night, Have Fun)
**Vad det är:** En CLI-körare som håller igång dina kodagenter autonomt tills du säger stopp — och gör en commit efter varje lyckad iteration.

**Varför det är intressant:** Har du kört Claude Code på en uppgift, gått iväg och kommit tillbaka till att agenten fastnat och ingenting hänt? GNHF löser det: den kör agenten i en loop, committar varje steg som fungerar, och skriver en `notes.md` med lägesrapporten om agenten behöver starta om från noll. Fungerar med Claude Code, Codex, GitHub Copilot CLI och flera andra — du väljer själv. Tanken är att du ger uppdraget på kvällen och hittar ett repo fullt av commits på morgonen.

**Buzz-nivå:** Håller på att bli en grej — 2 900 GitHub-stjärnor, 42 versioner, skaparen nådde 1 000 stjärnor "organiskt utan VC-pengar".

**Testa:** https://github.com/kunchenguid/gnhf — `npm install -g gnhf`, kör sedan `gnhf start` i ditt projekt.

### archify
**Vad det är:** En skill (tilläggsfunktion) till Claude Code, Codex och liknande agenter som omvandlar en klarspråksbeskrivning av ett system till ett professionellt arkitekturdiagram — exporterbart som SVG, PNG eller HTML.

**Varför det är intressant:** Vill du förklara hur ett system hänger ihop — för kollegor, kunder eller en AI-agent som ska arbeta med det — brukar du behöva Miro, Lucidchart eller manuell diagramsyntax. Archify gör det med en enda mening direkt i chatten: "Rita ett sekvensdiagram för hur betalningsflödet fungerar". Diagrammet renderas som en fristående HTML-fil och kan förfinas iterativt via chatt ("flytta autentiseringstjänsten till vänster"). Senaste versionen kom 5 juli 2026.

**Buzz-nivå:** Börjar fångas upp — 2 614 GitHub-stjärnor, aktiv veckoutveckling.

**Testa:** https://github.com/tt-a1i/archify — Installeras som en Claude Code skill via `claude skills install`.

### OpenKnowledge (Inkeep)
**Vad det är:** En lokal anteckningsapp med WYSIWYG-vy (du ser formaterad text medan du skriver, precis som Notion) — och en inbyggd MCP-server som låter Claude Code och Cursor söka i och skriva till dina anteckningar direkt.

**Varför det är intressant:** Obsidian saknar AI-agentintegration ur lådan; Notion kräver att dina anteckningar lagras i molnet. OpenKnowledge är local-first (allt sparas på din dator), öppen källkod och ger dina kodningsagenter direkt tillgång till din kunskapsbas — de kan söka bland dina anteckningar när de löser problem, utan att data lämnar maskinen.

**Buzz-nivå:** Börjar fångas upp — Show HN-post (27 juni 2026), 1 900 GitHub-stjärnor.

**Testa:** https://github.com/inkeep/open-knowledge — `npm install -g open-knowledge`, aktivera MCP-servern i inställningarna.

## Verktyg, automation & infrastruktur

### Craft Agents OSS
**Vad det är:** En öppen skrivbordsapp för att hantera AI-agenter med multi-session-inkorg, realtids-streaming och koppling mot GitHub, Linear och MCP-servrar — och en headless server för de som vill integrera det i egna system.

**Varför det är intressant:** Att arbeta med flera AI-agenter parallellt kräver normalt att man jonglerar terminaltabbar, chatthistorik och Git-loggar. Craft Agents OSS samlar allt i en vy med ett polerat grafiskt gränssnitt. Bygg av craft.do-teamet för eget internt agentarbete och sedan open-sourceat. Stödjer Anthropic, OpenAI, Google, GitHub Copilot och mer i ett och samma fönster.

**Buzz-nivå:** Håller på att bli en grej — 6 700 GitHub-stjärnor, senaste release 1 juli 2026.

**Testa:** https://github.com/craft-ai-agents/craft-agents-oss — Ladda ned app-paketet för Mac, Windows eller Linux.

### agent-orchestrator
**Vad det är:** En koordinatortjänst (kör i bakgrunden) som hanterar flera parallella AI-kodningsagenter — varje agent jobbar i sin egna isolerade kopia av projektet, och koordinatorn ser till att CI-fel och kodkonflikter hamnar hos rätt agent att lösa.

**Varför det är intressant:** När du kör fem agenter parallellt uppstår frågan: vem fixar det som går sönder? agent-orchestrator svarar automatiskt — det är den agent som bröt det, och uppgiften skickas tillbaka dit. Ger live terminaltillgång per agent, är PR-medveten och stödjer 23+ agentadaptrar (Claude Code, Cursor, Aider, Codex med flera).

**Buzz-nivå:** Håller på att bli en grej — 8 045 GitHub-stjärnor, senaste release 5 juli 2026.

**Testa:** https://github.com/AgentWrapper/agent-orchestrator — Go-daemon + Electron-UI, installeras med `go install`.

### CubeSandbox (Tencent Cloud)
**Vad det är:** En säker sandlåda för AI-agenters kodexekvering — varje agent kör i sin egna mini-VM med en separat operativsystemskärna, startar på under en sekund och kräver minimalt extra minne.

**Varför det är intressant:** När en AI-agent genererar och kör kod finns alltid en risk: vad händer om koden gör något oväntat? Vanliga containerlösningar (som Docker) delar kärna med din dator, vilket kan utnyttjas av skadlig kod. CubeSandbox använder verklig VM-isolering (samma teknik som vanliga virtuella datorer) men utan prestandastraffet. Open-sourceat av Tencent Cloud och kan byta ut E2B-sandboxar med en enda inställningsändring.

**Buzz-nivå:** Håller på att bli en grej — 7 303 GitHub-stjärnor, release 3 juli 2026.

**Testa:** https://github.com/TencentCloud/CubeSandbox — Konfigureras via Docker Compose, kompatibelt med E2B SDK.

### Deepsec (Vercel Labs)
**Vad det är:** Ett verktyg som skickar in AI-agenter (Claude Opus + Codex) för att självständigt leta igenom hela kodbaser efter säkerhetshål — och levererar fynd med bevisad exploatering, inte bara varningar.

**Varför det är intressant:** Traditionella säkerhetsskanners (Snyk, Semgrep) letar efter kända mönster och genererar massor av falsklarm. Deepsec använder agenter med djupare resonemang som faktiskt spårar hur data flödar genom koden för att avgöra om en sårbarhet är exploaterbar på riktigt. Det minskar brus och ökar förtroendet för fynden. Körs lokalt, inga molnberoenden.

**Buzz-nivå:** Håller på att bli en grej — 5 100 GitHub-stjärnor, +1 800 under senaste veckan (5 juli 2026).

**Testa:** https://github.com/vercel-labs/deepsec — `npx deepsec init` i roten på ditt projekt.

### Journi DevOS
**Vad det är:** En plattform som ger organisationer insyn i hur Claude Code, Cursor och andra AI-kodningsverktyg faktiskt används — session för session — och beräknar kostnader, tidsvinster och ROI.

**Varför det är intressant:** De flesta som kör AI-kodningsverktyg i team har ingen aning om hur de används, var de misslyckas eller vad de faktiskt levererar i produktivitetsvinst. DevOS instrumenterar varje session och ger managers exporterbar analys. Kan driftsättas helt inom den egna miljön utan att data skickas utanför.

**Buzz-nivå:** Extremt tidigt — lanserat juni 2026, täckt i branschpress (MartechSeries).

**Testa:** https://devos.team/ — Kontakta för pilot.

### Dawnguard
**Vad det är:** En säkerhetsplattform som fångar molnarkitekturbrister vid designbordet — innan någon enda rad kod är skriven — och automatiskt översätter godkänd design till driftsättningsbar infrastruktur.

**Varför det är intressant:** Konkurrenter som Snyk och Wiz granskar kod och infrastruktur som redan finns. Dawnguard arbetar ett steg tidigare: en delad arkitekturarbetsyta (tänk Miro för molndesign) som i realtid validerar varje beslut mot säkerhetsregler och bästa praxis. Målet är att säkerhetsproblem aldrig ska ens hinna uppstå i koden.

**Buzz-nivå:** Extremt tidigt — lanserat 1 juli 2026, $3,3M ny finansieringsrunda (totalt $6,3M, lett av BNVT Capital).

**Testa:** https://dawnguard.ai/ — Boka demo, tidig beta.

## Specialgrejer & experiment

### Flash-MoE
**Vad det är:** En minimalistisk körmotor (skriven i ren C och Apples Metal-grafik-API) som kör en av världens största öppna AI-modeller — 400 miljarder parametrar — direkt från hårddisken på en vanlig MacBook Pro, utan moln och utan GPU-kluster.

**Varför det är intressant:** En 400-miljardersparametersmodell kräver normalt ett datacenter med specialiserade GPUer. Flash-MoE visar att det faktiskt går att köra den lokalt genom att strömma modellen bit för bit från SSD:n när den behövs — en teknik inspirerad av hur spelmotorer laddar spelvärldar. Noll beroenden utöver Xcode. Byggt på 24 timmar med Claude Code av en ensam utvecklare, och inspirerade omedelbart minst fyra community-forks med egna optimeringar.

**Buzz-nivå:** Börjar fångas upp — Hacker News front page kort efter lansering, minst fyra oberoende community-forks indexerade inom dagar.

**Testa:** https://github.com/danveloper/flash-moe — Kräver Mac med Apple Silicon och minst 48 GB RAM.

### jcode
**Vad det är:** En snabb, resurssnål kodningsagent-körmotor skriven i Rust — använder 27 MB minne jämfört med 330+ MB för de flesta alternativ, och stödjer parallella agentgrupper som automatiskt löser konflikter sinsemellan.

**Varför det är intressant:** Riktar sig mot de som vill köra AI-kodningsagenter på lokala modeller utan att systemet hänger sig. Stödjer Claude, OpenAI, Gemini och GitHub Copilot men är optimerat för tung lastning. 107 versioner på kort tid visar aktiv och snabb iteration. MIT-licensierat individuellt projekt.

**Buzz-nivå:** Håller på att bli en grej — 8 147 GitHub-stjärnor (senast 4 juli 2026).

**Testa:** https://github.com/1jehuang/jcode — `cargo install jcode`, konfigureras via en TOML-fil.

---

### Astryx (Meta)
**Vad det är:** Metas interna React-komponentbibliotek (150+ komponenter, använt i 13 000+ interna appar) — nu open-sourceat med en CLI och MCP-server specifikt byggd för att AI-agenter ska kunna bygga grafiska gränssnitt utan att gissa sig fram.

**Varför det är intressant:** Det unika är ansatsen: istället för att du konfigurerar designen ger Astryx agenter ett "manifest" som beskriver exakt vilka byggklossar som finns, deras typer och vad de returnerar — ungefär som ett API-dokument fast för UI-komponenter. Det gör att agenter kan generera konsistenta gränssnitt utan att behöva ta stilbeslut varje gång.

**Buzz-nivå:** Håller på att bli en grej — 5 400 GitHub-stjärnor, +4 760 under en vecka sedan lansering 27 juni 2026.

**Testa:** https://github.com/facebook/astryx — `npx create-astryx-app` för att starta ett nytt projekt.
