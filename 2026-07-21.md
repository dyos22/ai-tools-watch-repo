# AI-verktygsbevakning: 2026-07-21

## Inledning

Veckans tema: **MCP mognar — och ekosystemet runt det exploderar.** Det protokoll som låter AI-agenter använda externa verktyg (Model Context Protocol, uppfunnet av Anthropic) har gått från experiment till infrastruktur. Nu börjar ett helt lager av stödverktyg växa fram runt det: gateways, säkerhetsplattformar, React-ramverk, aggregatorer. Det liknar hur webbens ekosystem en gång byggdes kring HTTP. Parallellt dyker en kinesisk konkurrent till Claude Code upp med inbyggd parallellism och videostöd, en YC-startup ger AI-agenter egna e-postadresser, och ett säkerhetsverktyg gör äntligen något konkret åt problemet att kodningsagenter ibland råkar radera fel saker.

## Topp 3 att testa direkt

**Playwriter** — Låter din AI-agent använda din riktiga Chrome-webbläsare med alla dina inloggningar intakta, istället för en ny tom instans utan access. Det löser det praktiska problemet att agenter inte kan komma åt sidor som kräver inloggning. (se *Verktyg, automation & infrastruktur*)

**destructive_command_guard** — Sätter ett osynligt skyddsnät under Claude Code och andra kodningsagenter och fångar farliga kommandon — "radera alla filer", "nollställ git-historiken" — innan de faktiskt körs. (se *Gör Claude Code och andra AI-assistenter smartare*)

**AgentMail** — Ger AI-agenter egna e-postadresser; inte AI-hjälp med *din* mail, utan riktiga brevlådor som ägs och drivs av agenten självständigt. YC-startup med ovanligt konkret användningsfall. (se *Verktyg, automation & infrastruktur*)

## Gör Claude Code och andra AI-assistenter smartare

### destructive_command_guard

**Vad det är:** En liten, extremt snabb vaktpost som sitter mellan din AI-kodningsagent och din dator och blockerar farliga kommandon innan de körs — utan att du märker av fördröjningen.

**Varför det är intressant:** Claude Code, Cursor, Codex och liknande agenter kan av misstag köra kommandon som är svåra att ångra — "ta bort hela src-mappen", "återställ git-historiken", "radera databastabellen". Verktyget skannar till och med dolda kommandon i skript och inkluderar färdiga skyddsprofiler för AWS, GCP, Kubernetes, PostgreSQL och Docker. Skillnaden mot generiska säkerhetsskal: det är byggt specifikt för eran med AI-kodningsagenter och integrerar direkt i alla större agentmiljöer.

**Buzz-nivå:** Håller på att bli en grej — 5 200 GitHub-stjärnor (21 juli 2026), varav 1 400 på en enda vecka efter att en säkerhetsartikel i The Hacker News lyfte projektet.

**Testa:** https://github.com/Dicklesworthstone/destructive_command_guard — Koppla till Claude Code som ett PreToolUse-hook i din konfigurationsfil.

### agentic-tools-mcp

**Vad det är:** En liten server som ger din AI-assistent ett ihärdigt uppgiftssystem och ett minne som faktiskt överlever mellan sessioner.

**Varför det är intressant:** Claude Code "glömmer" allting varje gång du stänger terminalen. Med detta MCP-tillägg får agenten ett strukturerat att-göra-system med obegränsad djupnivå — uppgifter, deluppgifter, del-deluppgifter, prioritet, beroenden och tidsspårning — allt sparat som vanliga filer i projektet som går att versionera med git och dela med ett team. Inkluderar också ett minne med relevansbedömning som agenten kan söka i. En VS Code-extension ger ett grafiskt gränssnitt ovanpå.

**Buzz-nivå:** Extremt tidigt — 87 GitHub-stjärnor (21 juli 2026), men listad som ett av de 15 bästa MCP-servrarna för kodning av Taskade.

**Testa:** https://github.com/Pimzino/agentic-tools-mcp — `npm install -g agentic-tools-mcp`, konfigurera sedan i Claude Desktop eller Claude Code.

### kimi-code (MoonshotAI)

**Vad det är:** En terminal-baserad AI-kodningsagent från det kinesiska AI-labbet MoonshotAI — en direkt konkurrent till Claude Code med inbyggda parallella agenter och stöd för videoinmatning.

**Varför det är intressant:** Där Claude Code kräver extra verktyg för att köra saker parallellt levererar kimi-code tre inbyggda agentroller (kodare, utforskare, planerare) som samarbetar direkt ur lådan. Tar dessutom emot skärminspelningar som kontext — du kan visa agenten exakt vad som gick fel visuellt, vilket ingen annan kodningsagent gör. Distribueras som en enda binärfil utan externa beroenden. Signalerar att kinesiska AI-labb nu bygger seriös verktygskjede, inte bara modeller.

**Buzz-nivå:** Håller på att bli en grej — 4 200 GitHub-stjärnor (21 juli 2026) sedan lanseringen i juni 2026, med 52 releaser på sex veckor.

**Testa:** https://github.com/MoonshotAI/kimi-code — Fungerar med Kimi K2.5-modellen; gratis tier tillgängligt.

### idea-reality-mcp

**Vad det är:** En liten MCP-server som låter din AI-agent kontrollera om din startup-idé eller funktion redan existerar — innan den börjar bygga.

**Varför det är intressant:** En vanlig fälla vid AI-assisterad kodning är att man bygger något som redan finns som ett polerat open source-paket. idea-reality-mcp söker automatiskt igenom GitHub, Hacker News, npm, PyPI och Product Hunt och returnerar en 0–100-poäng ("reality score") med bevis, konkurrentlista och trendanalys (accelererande/stabil/minskande). Kopplas till Claude Code med ett enda kommando.

**Buzz-nivå:** Börjar fångas upp — 761 GitHub-stjärnor (21 juli 2026), #1 i Product Hunt AI Digest 1 juli 2026.

**Testa:** https://github.com/mnemox-ai/idea-reality-mcp — `claude mcp add idea-reality -- uvx idea-reality-mcp`

## Verktyg, automation & infrastruktur

### Playwriter

**Vad det är:** En Chrome-extension plus kommandoradsverktyg som låter AI-agenter styra din *redan inloggade* webbläsare — med alla dina sessioner och cookies intakta.

**Varför det är intressant:** Standardlösningen för att låta AI-agenter surfa (till exempel Microsoft Playwright-MCP) öppnar en helt ny, tom webbläsare utan dina inloggningar, utan dina cookies och utan access till banktjänster eller OAuth-flöden. Playwriter kopplar istället direkt till din körande Chrome-instans. Resultatet: agenten kan utföra uppgifter på sidor du redan är inloggad på — faktiska arbetsflöden i stället för sandlådetest. Fungerar med Claude Desktop, Cursor och Windsurf.

**Buzz-nivå:** Håller på att bli en grej — 3 700 GitHub-stjärnor (21 juli 2026), aktivt diskuterat i Claude- och LocalLLaMA-communityt för bank- och OAuth-automation.

**Testa:** https://github.com/remorses/playwriter — Installera extension + `npx playwriter`.

### AgentMail

**Vad det är:** Ett API som skapar riktiga e-postadresser som ägs av och är avsedda för AI-agenter — inte AI som hjälper dig med din e-post, utan brevlådor som agenten kontrollerar helt självständigt.

**Varför det är intressant:** Alla befintliga e-post-AI-verktyg hjälper *dig* med *din* inkorg. AgentMail vänder på det: varje agent får en egen `@agentmail.to`-adress, kan ta emot och skicka mejl, söka i konversationstrådar och hantera bilagor via ett enkelt API med realtidsnotiser. Möjliggör flöden som en kundsupportagent med egen svarsadress, eller en rekryteringsagent med isolerad utkorgsadress — utan att blanda in din personliga inbox. YC W26-startup.

**Buzz-nivå:** Börjar fångas upp — 500+ affärskunder (juli 2026), YC W26, 6 miljoner dollar i seed-finansiering med investerare som Paul Graham (mars 2026).

**Testa:** https://www.agentmail.to — Gratis tier med 100 e-postmeddelanden per månad.

### MetaMCP (metatool-ai)

**Vad det är:** En lokal Docker-baserad samlingspunkt som aggregerar alla dina MCP-servrar till ett enda konfigurerat gränssnitt med grafisk panel, namnrymder och inbyggd autentisering.

**Varför det är intressant:** När man väl börjat använda MCP-verktyg med Claude — filsystem, GitHub, databaser, webbläsare, API:er — exploderar konfigurationskomplexiteten snabbt. MetaMCP fungerar som en omvandlare: i stället för att konfigurera varje server separat i varje AI-klient pekar du alltihop mot MetaMCP, grupperar dem i namngivna arbetsytor, och byter hela konfigurationer med ett klick. Red Hat Developer-bloggen lyfter det för storskaliga driftsättningar.

**Buzz-nivå:** Håller på att bli en grej — 2 500 GitHub-stjärnor (21 juli 2026), omnämnd i Red Hat Developer-bloggen april 2026.

**Testa:** https://github.com/metatool-ai/metamcp — `docker run metamcp/metamcp`, öppna det grafiska gränssnittet på localhost.

### ToolHive (Stacklok)

**Vad det är:** En Kubernetes-baserad plattform för att köra MCP-servrar i isolerade containrar med företagssäkerhetspolicyer, revisionsloggar och ett verifierat verktygsregister.

**Varför det är intressant:** De flesta MCP-servrar körs utan isolation — råa API-nycklar i konfigurationsfiler, ingen åtkomstkontroll, inga loggar. ToolHive behandlar MCP-servrar som processer i containrar: var och en körs isolerat, en gateway verkställer identitetskontroll och policyer, och ett register säkerställer att du vet var koden kom ifrån. Grundat av Craig McLuckie, en av skaparna av Kubernetes. Fyller ett kritiskt gap i takt med att MCP går från hobbyexperiment till produktionsmiljö.

**Buzz-nivå:** Börjar fångas upp — 1 900 GitHub-stjärnor (21 juli 2026), aktivt i cloudnative-communityt.

**Testa:** https://github.com/stacklok/toolhive — `thv run --transport=sse filesystem` startar en isolerad filsystem-MCP-server.

### Rowboat

**Vad det är:** En lokal, öppen källkods-app som indexerar din e-post, dina möten och dina Slack-meddelanden till ett kunskapsnätverk och sedan kör bakgrundsagenter på schemalagda triggers.

**Varför det är intressant:** Rowboat är inte ytterligare ett chat-gränssnitt — det bygger ett levande, Obsidian-kompatibelt kunskapsnätverk av din faktiska arbetskontext (Gmail, Kalender, Granola, Fireflies) och låter bakgrundsagenter triggas av nya e-postmeddelanden eller tidsscheman. Agenterna kan surfa på webben, skriva kod via Claude Code eller Codex, och anropa MCP-anslutna verktyg. Allt körs lokalt; dina data lämnar inte din dator. Saknar direkta konkurrenter — är varken Notion AI, Devin eller en fristående assistent, utan lite av allt tre.

**Buzz-nivå:** Extremt tidigt — 112 GitHub-stjärnor (21 juli 2026), men syntes på GitHub Trending veckan 12 juli.

**Testa:** https://github.com/rowboatlabs/rowboat — Kör lokalt på macOS/Linux, kräver Docker.

### Skybridge (alpic-ai)

**Vad det är:** Ett TypeScript/React-ramverk för att bygga interaktiva appar som körs inuti Claude, ChatGPT och VS Code via MCP — tänk "React för MCP-appar."

**Varför det är intressant:** MCP ger agenter verktyg, men det har saknats ett standardsätt att bygga faktiskt interaktivt gränssnitt inuti de verktygskontexterna. Skybridge abstraherar bort skillnaderna mellan MCP-klienter (Claude vs. ChatGPT vs. VS Code), levererar typsäkra komponenter, varm-omladdning och en lokal emulator. Skriv en gång, driftsätt i hela MCP-ekosystemet. Omnämnt i OpenAI-dokumentation som referensimplementation.

**Buzz-nivå:** Håller på att bli en grej — 1 900 GitHub-stjärnor, 100 000 npm-nedladdningar per månad (21 juli 2026).

**Testa:** https://github.com/alpic-ai/skybridge — `npx create-skybridge-app`.

## Specialgrejer & experiment

### Statewright

**Vad det är:** En Rust-baserad motor som begränsar vilka verktyg en AI-agent får tillgång till i varje fas av ett arbetsflöde — som trafikljus för vad agenten får göra, men på infrastrukturnivå.

**Varför det är intressant:** Det vanligaste sättet att göra AI-agenter mer pålitliga är att skriva bättre instruktioner — och det fungerar bara delvis. Statewright löser det istället strukturellt: du definierar ett visuellt flödesschema (en tillståndsmaskin) som styr exakt vilka MCP-verktyg agenten kan använda i varje steg. Agenten kan inte göra mer än vad schemat tillåter, oavsett vad instruktionerna säger. Integreras med Claude Code, Codex och Cursor. I ett tidigt test på tio uppgifter gick framgångsgraden från 2/10 till 10/10 med Statewright-begränsningar.

**Buzz-nivå:** Extremt tidigt — Show HN-inlägg med ~120 röster, Apache 2.0 open source.

**Testa:** https://github.com/statewright/statewright — https://statewright.ai/

### VulnHunter (Capital One)

**Vad det är:** Ett öppen källkods-AI-verktyg som skannar din källkod som en *angripare* skulle — spårar exploateringsvägar från nätverksgränsen inåt och föreslår riktade fixar.

**Varför det är intressant:** Traditionella säkerhetsskannrar (Snyk, Semgrep) letar efter kända mönster och producerar massor av falska positiver. VulnHunter resonerar som en faktisk angripare: börjar vid nätverksåtkomliga ingångspunkter och spårar möjliga attackvägar logiskt framåt. En inbyggd "falsifikationsmotor" försöker sedan aktivt motbevisa sina egna slutsatser för att rensa bort falskt alarm. Körs som en Claude-agent men är modell-agnostisk. Lanserat av Capital Ones säkerhetsteam 17 juli 2026.

**Buzz-nivå:** Extremt tidigt — precis lanserat (17 juli 2026), uppmärksammat av VentureBeat och SecurityWeek direkt vid lansering.

**Testa:** https://github.com/capitalone/vulnhunter — Kör som ett pre-commit-steg eller en fristående agent i din CI-pipeline.
