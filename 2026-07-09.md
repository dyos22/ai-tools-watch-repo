# AI-verktygsspaning 2026-07-09

## Inledning

Den här veckan domineras spaningen av ett tema: enklare agenter som gör mer. Från en 100-raders kodningsagent som slår kommersiella alternativ på riktiga buggreparationer till ett virtuellt filsystem där AI-agenter läser från S3, Slack och Gmail som vanliga mappar — kraven på komplexitet sjunker medan förmågan ökar. Fem fynd handlar om att göra Claude Code billigare och smartare att köra; resten rör ny infrastruktur för agenter som faktiskt ska fungera i produktionsmiljöer. 13 fynd totalt.

## Gör Claude Code och andra AI-assistenter smartare

### Tokenwise
**Vad det är:** En LLM-proxy — ett mellansteg mellan din kod och AI-APIet — som ger realtidsvy över kostnader, svarstider och kvalitet per anrop, plus A/B-routing för att testa nya modeller utan att ändra din kod.

**Varför det är intressant:** Alla som kör AI-agenter eller appar med ett AI-API möter samma problem: det är svårt att veta vad som kostar pengar och varför. Tokenwise löser det med en enda radändring — byt bara ut den URL din kod skickar anrop till. Sedan kan du dela trafik ("10 % av anropen går till den nya modellen, resten till den gamla") och mäta kvaliteten automatiskt innan du byter fullt ut. Enklare att komma igång med än specialverktyg som LiteLLM eller Portkey för små team.

**Buzz-nivå:** Extremt tidigt — Product Hunt-lansering 1 juni 2026, gratis att starta utan kreditkort, omnämnd i AI Weekly.

**Testa:** https://tokenwisehq.com/ — registrera dig och byt `base_url` i din OpenAI-klient till Tokenwise-proxyn.

### LeanCTX
**Vad det är:** En lokal programfil (skriven i Rust) som sitter mellan dig och Claude Code och styr vad agenten faktiskt läser — och komprimerar kontexten kraftigt innan den skickas.

**Varför det är intressant:** Claude Code läser ibland hela filer när den bara behöver titta på strukturen. LeanCTX introducerar ett "karta-läge": istället för att läsa 50 filer i sin helhet (ca 533 000 tecken) får agenten en komprimerad strukturöversikt (ca 8 000 tecken) — 98 % reduktion. Den komprimerar dessutom systeminstruktioner och konversationshistorik utan att bryta inbyggd cachning. 76 MCP-verktyg, stöd för Claude Code, Cursor och Gemini CLI.

**Buzz-nivå:** Börjar fångas upp — 3 200 GitHub-stjärnor, nästintill dagliga uppdateringar sedan mars 2026.

**Testa:** https://github.com/yvgude/lean-ctx — ladda ned binären och konfigurera som MCP-server i Claude Codes inställningsfil.

### Dirac
**Vad det är:** En AI-kodningsagent (VSCode-tillägg + terminalkommando) som gör kodändringar 50–80 % billigare än alternativ tack vare ett smartare sätt att peka ut exakt var i koden en ändring ska göras.

**Varför det är intressant:** Vanliga AI-agenter refererar till rader med radnummer ("ändra rad 42"), men om du lagt till rader ovan pekar de sedan fel. Dirac använder istället ett unikt fingeravtryck per kodrad — ändringspekar korrekt även om filen förändrats sedan agenten senast läste den. Tog andraplatsen på Terminal-Bench-2-rankingen med 65,2 % korrekthet på Gemini 3 Flash (Googles egna baseline låg på 47,6 %).

**Buzz-nivå:** Börjar fångas upp — 1 400 GitHub-stjärnor, diskuterades på Hacker News efter publicering.

**Testa:** https://github.com/dirac-run/dirac — installera VSCode-tillägget eller kör `dirac init` i ett projekt.

### TencentDB Agent Memory
**Vad det är:** Ett lokalt, helt offlinekapabelt långtidsminne för AI-agenter med ett fyrnivåsystem — råa konversationer → atomfakta → scenarion → personaprofil.

**Varför det är intressant:** De flesta minneslösningar för AI-agenter kastar allt i en enda hög. Det här systemet sorterar automatiskt: atomfakta ("du föredrar Python"), scenarion ("hur du brukar felsöka nätverksproblem"), och en personaprofil som uppdateras dagligen — du behöver aldrig återintroducera dig i varje ny session. Lokal SQLite-databas, inga externa tjänster, inbyggd hybridsökning (nyckelord + semantisk likhet). Rapporterade 51 % bättre uppgiftsframgång jämfört med agenter utan minne.

**Buzz-nivå:** Håller på att bli en grej — 7 800 GitHub-stjärnor, nådde GitHub Trending med 318 nya stjärnor på en dag.

**Testa:** https://github.com/TencentCloud/TencentDB-Agent-Memory — `pip install tencent-agent-memory`, konfigurera som MCP-server.

### MCP-specifikationens Release Candidate (annonserad)
**Vad det är:** En stor uppdatering av MCP — det protokoll som låter Claude Code och andra AI-verktyg kommunicera med externa tjänster — med spec-lock annonserad till 28 juli 2026.

**Varför det är intressant:** Den viktigaste förändringen: MCP-servrar behöver inte längre hålla reda på vilken "session" varje anrop tillhör. Det låter tekniskt, men innebär i praktiken att MCP-verktyg nu kan driftas bakom vanlig molninfrastruktur (standardlastbalansering) utan speciallösningar — billigare och mer pålitligt. Protokollet får också ett nytt sätt att visa gränssnittselement i sandboxade fönster. Tre äldre funktioner fasas ut med tolv månaders varsel, vilket ger ekosystemet tid att anpassa sig.

**Buzz-nivå:** Ekosystemnyheter — officiell annons från Model Context Protocol-teamet inför spec-lock 28 juli.

**Testa:** https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/ — läs förändringarna och uppdatera dina MCP-servrar inför spec-lock.

## Verktyg, automation & infrastruktur

### mini-swe-agent
**Vad det är:** En AI-kodningsagent i 100 rader Python från SWE-bench-teamet (Princeton/Stanford) som löser GitHub-problem via rena terminalkommandon — ingen konfiguration, inga komplicerade inställningar.

**Varför det är intressant:** De flesta moderna AI-kodningsagenter är komplexa system med stora kodbasers. mini-swe-agent demonstrerar att enkelt ibland slår komplicerat: 74 % rätt på SWE-bench Verified — bättre än flera kommersiella alternativ — med en agent som ryms i en enda kort fil. Används som utgångspunkt av team på Meta, NVIDIA och IBM som vill bygga anpassade egna agenter utan att ta över en hel kommersiell plattform.

**Buzz-nivå:** Håller på att bli en grej — 5 700 GitHub-stjärnor, forks från stora teknikbolag.

**Testa:** https://github.com/SWE-agent/mini-swe-agent — `pip install mini-swe-agent` och kör `mini-swe-agent --issue <GitHub-länk>`.

### Tabularis
**Vad det är:** En gratis, open source desktop-app för att arbeta med databaser (PostgreSQL, MySQL, SQLite, DuckDB, ClickHouse med flera) med inbyggd MCP-server för Claude, Cursor och Devin.

**Varför det är intressant:** Populära databasverktyg som DBeaver och TablePlus saknar AI-integration. Tabularis aktiverar MCP-servern med ett enda kommando, vilket låter AI-agenter lista dina databasuppkopplingar, inspektera tabellscheman och köra SQL-frågor direkt. Stöder dessutom SQL-anteckningsböcker (Markdown och SQL i samma dokument) och ett visuellt diagram som visar varför en databasfråga är långsam. Gratis, Apache 2.0, ingen betalfunktion.

**Buzz-nivå:** Börjar fångas upp — 3 600 GitHub-stjärnor, 236 forks sedan januari 2026.

**Testa:** https://github.com/TabularisDB/tabularis — ladda ned desktop-appen, kör `tabularis --enable-mcp` för att aktivera AI-integrationen.

### Camofox Browser
**Vad det är:** En specialbyggd webbläsare för AI-agenter, baserad på Firefox, som kringgår blockeringen som stoppar agenter på moderna webbplatser.

**Varför det är intressant:** AI-agenter som automatiserar webbsurfning (via Playwright eller liknande) blockeras ofta av Cloudflare och andra bot-skyddsystem. Orsaken: webbläsarens "fingeravtryck" avslöjas av JavaScript som körs efter att sidan laddats. Camofox åtgärdar detta på ett djupare plan — fingeravtrycket modifieras i webbläsarens C++-kod innan JavaScript ens startar, vilket gör agenten omöjlig att skilja från en riktig användare. Rapporteras ge 90 % lägre tokenanvändning jämfört med vanlig HTML-hämtning. Kompatibel med befintlig Playwright-kod utan omskrivning.

**Buzz-nivå:** Håller på att bli en grej — 7 500 GitHub-stjärnor, 778 forks.

**Testa:** https://github.com/jo-inc/camofox-browser — starta via Docker och anslut med befintlig Playwright-kod.

### Mirage
**Vad det är:** Ett virtuellt filsystem som monterar S3, Google Drive, Slack, Gmail, GitHub, PostgreSQL, Redis och ~50 andra tjänster sida vid sida som om de vore vanliga mappar i ett Unix-skal.

**Varför det är intressant:** En AI-agent som behöver data från tre olika tjänster behöver normalt tre API-bibliotek och tre separata inloggningar. Med Mirage kan agenten istället köra `ls /slack/channels/general` eller `cat /s3/hink/rapport.json` som om det vore lokala filer — och flytta data direkt mellan tjänster: `cat /github/issues.json | /gmail/skicka`. Integrerar utan extrauppsättning med OpenAI Agents SDK, Vercel AI SDK och LangChain.

**Buzz-nivå:** Börjar fångas upp — växte från 714 till 3 300 GitHub-stjärnor maj–juli 2026.

**Testa:** https://github.com/strukto-ai/mirage — `pip install mirage-fs` och konfigurera API-nycklar per backend.

### Yao
**Vad det är:** En enda liten körbar fil (skriven i Go) som fungerar som komplett körmiljö för AI-agenter och webbappar, med inbyggt MCP-stöd och ett integrerat 4-nivåers minnessystem — nu i version 1.0 Release Candidate.

**Varför det är intressant:** Att sätta upp ett agentprojekt innebär normalt att installera många Python-paket med motstridiga beroenden. Yao löser det med en enda binär utan externa beroenden: MCP-stöd via tre kommunikationssätt, ett inbyggt JavaScript-läge för logik, och minne per anrop/session/användare/team. Enkelbinär-modellen gör det radikalt enklare att driftsätta agenter i produktionsmiljöer — kopiera filen, starta. Milstolpen v1.0.0-rc2 markerar att API:t nu anses stabilt nog för produktion.

**Buzz-nivå:** Håller på att bli en grej — 7 500 GitHub-stjärnor, v1.0.0-rc2 lanserades 5 juli 2026.

**Testa:** https://github.com/YaoApp/yao — `curl -fsSL https://yao.app/install | bash`, sedan `yao start`.

### Astron RPA
**Vad det är:** En open source desktop-automationsplattform från iFlyTek med 300+ färdiga automatiseringsblock och dubbelriktad koppling till AI-agenter.

**Varför det är intressant:** Traditionell RPA (som UiPath) och AI-agenter lever vanligen i separata världar. Astron skapar en brygga: en AI-agent kan starta ett RPA-arbetsflöde ("öppna Excel, kopiera de här cellerna"), och ett RPA-arbetsflöde kan kalla på AI-agenten för beslut mitt i körningen. Den dubbelriktade kopplingen möjliggör automatiseringsflöden som blandar strukturerade uppgifter (RPA) med ostrukturerade beslut (AI) utan att kräva manuellt ingrepp. Self-hosted, fungerar på Windows och i Docker, ingen enterprise-prislapp.

**Buzz-nivå:** Håller på att bli en grej — 5 200 GitHub-stjärnor, 582 forks, senaste release v1.1.6 februari 2026.

**Testa:** https://github.com/iflytek/astron-rpa — ladda ned från GitHub Releases (Windows .exe), starta Astron Designer för att bygga flöden.

### Specific.dev
**Vad det är:** En molnplattform byggd specifikt för AI-kodningsagenter — "AWS för coding agents" (YC W26) — där agenten hanterar hela cykeln från kod till driftsättning utan att du behöver konfigurera infrastruktur.

**Varför det är intressant:** Det klassiska problemet: coding agents kan skriva och ändra kod, men deployment-konfiguration (molntjänster, miljövariabler, nätverk) kräver fortfarande manuellt arbete. Specific låter agenten skriva hela infrastrukturkonfigurationen som en granskningsbar fil (`specific.hcl`) — du kan läsa och godkänna den precis som kod. `specific dev` ger lokal miljö direkt, `specific deploy` sätter i produktion på minuter. Inget krav på att ge agenten fri tillgång till ditt molnkonto.

**Buzz-nivå:** Extremt tidigt — YC W26-batch, tidig access-lansering pågår.

**Testa:** https://specific.dev/ — ansök om tidig access.

## Specialgrejer & experiment

### Ghidra MCP
**Vad det är:** En MCP-server med 256 verktyg som kopplar Claude och andra AI-modeller direkt till Ghidra — NSA:s gratis program för att analysera kompilerade program vars källkod man inte har tillgång till.

**Varför det är intressant:** Säkerhetsforskare som letar sårbarhet i stängd mjukvara måste normalt kopiera ut maskinkod för hand in i en AI-chatt, ett steg i taget. Ghidra MCP låter en agent utföra hela analysen i ett svep: dekompilera funktioner, bygga anropsgrafer, matcha kodsignaturer mellan versioner och skriva tillbaka namngivning till projektet — med 93 % färre separata API-anrop via batchoperationer. En av de mer imponerande MCP-integrationerna för ett specialiserat professionellt domänverktyg.

**Buzz-nivå:** Börjar fångas upp — 2 700 GitHub-stjärnor, 35 releaser (senaste v5.14.2, 27 juni 2026).

**Testa:** https://github.com/bethington/ghidra-mcp — kräver Ghidra-installation, konfigurera sedan MCP-servern mot Claude.
